# 智慧园区与智慧楼宇一体化平台

> 人、车、物、能、安的统一运营底座
> 适用：产业园区 / 办公楼宇 / 工业园区 / 政府园区

---

## 1. 园区常见"烟囱系统"问题

### 系统割裂
- **门禁系统**：人脸识别、刷卡，数据不共享
- **停车系统**：车牌识别、收费，独立管理
- **安防监控**：摄像头、告警，无法联动
- **能耗管理**：水电气，数据分散
- **工单系统**：报修、巡检，流程断裂
- **访客系统**：登记、审批，与门禁不联动

### 数据孤岛
- **口径不一致**：
  - 门禁说在线1000人，能耗系统说800人
  - 停车说300辆车，实际在场只有250辆

- **无法形成运营闭环**：
  - 安防发现问题→无法自动派工单
  - 能耗异常→不知道是哪个设备/区域
  - 设备故障→维保记录找不到

- **决策缺乏依据**：
  - 园区运营成本不清晰
  - 资产使用效率不可见
  - 安全风险无法量化

### 处置链条长
- **事件响应慢**：
  - 从发现问题到派单平均2小时
  - 从派单到处置完成平均1天
  - 验收与复盘环节缺失

- **追责困难**：
  - 谁发现的？何时发现的？
  - 谁处理的？如何处理的？
  - 问题是否复发？根因是什么？

## 2. 平台定位

### 三个统一
- **统一数据底座**：
  - 所有子系统数据汇聚
  - 统一数据标准与口径
  - 形成园区数字孪生

- **统一事件中心**：
  - 告警、工单、巡检统一入口
  - 处置流程标准化
  - 全生命周期管理

- **统一运营看板**：
  - 一屏掌控园区全局
  - 多维度分析决策
  - 移动端随时查看

### 以"事件"为核心串联业务
```
事件来源 → 事件中心 → 处置派发 → 闭环验收 → 数据沉淀 → 持续优化
```

- **事件来源**：
  - 安防告警（入侵、聚集、烟火等）
  - 设备告警（故障、离线、异常）
  - 能耗告警（超标、异常波动）
  - 人工报修（电话、APP、现场）
  - 巡检发现（机器人、人工）

- **事件处置**：
  - 自动分级（P0/P1/P2/P3）
  - 自动派单（根据类型、区域、专业）
  - 流程跟踪（接单→处理→验收→归档）

- **价值闭环**：
  - 统计分析（高发类型、区域、时段）
  - 预防性维护（根据历史数据预测）
  - 持续优化（SLA管理、效率提升）

## 4. 统一事件中心

### 事件模型
```json
{
  "event_id": "EVT-20260209-001234",
  "event_type": "alarm/fault/repair/inspection",
  "event_source": "video_analysis/sensor/manual",
  "severity": "P0/P1/P2/P3",
  "title": "1号楼3楼消防通道堵塞",
  "description": "AI视频分析检测到消防通道有杂物堆放",
  "location": {
    "building": "1号楼",
    "floor": "3F",
    "room": "消防通道",
    "coordinates": [120.123, 30.456]
  },
  "asset_id": "CAM-001",
  "create_time": "2026-02-09 14:30:00",
  "update_time": "2026-02-09 14:35:00",
  "status": "pending/processing/resolved/closed",
  "assigned_to": "张三",
  "related_ticket": "WO-20260209-001",
  "attachments": ["image1.jpg", "video1.mp4"]
}
```

### 事件生命周期
```
创建 → 待派发 → 已派发 → 处理中 → 待验收 → 已验收 → 已归档
```

- **自动派发规则**：
  - 根据事件类型自动分配专业（电工/水暖/保洁/保安）
  - 根据位置就近分配（同楼层/同区域优先）
  - 根据负载均衡（避免某人任务过多）
  - 紧急事件立即通知（P0级电话+短信+APP）

- **超时升级**：
  - P0级：30分钟未响应自动升级至主管
  - P1级：2小时未响应自动升级
  - P2级：4小时未响应自动升级
  - P3级：8小时未响应自动升级

- **验收机制**：
  - 必须上传现场照片/视频
  - 验收不通过自动回退重新处理
  - 验收人可以是创建人或管理员

## 5. 工单闭环管理

### 工单类型
- **报修工单**：设备故障、设施损坏
- **巡检工单**：定期巡检、专项检查
- **保洁工单**：日常保洁、深度清洁
- **安防工单**：异常处置、事件调查
- **能耗工单**：能耗异常排查、节能改造

### 工单流程
```
创建 → 审批（可选）→ 派发 → 接单 → 处理 → 验收 → 归档 → 统计分析
```

### 移动端支持
- **工单接收**：APP/小程序实时推送
- **位置导航**：一键导航到现场
- **语音录入**：语音转文字记录处理过程
- **拍照上传**：现场照片/视频实时上传
- **扫码报修**：扫设备二维码自动创建工单

### SLA管理
- **时效指标**：
  - 响应时长：从派发到接单
  - 处理时长：从接单到处理完成
  - 验收时长：从完成到验收通过

- **质量指标**：
  - 一次通过率：验收一次通过比例
  - 返工率：验收不通过需返工比例
  - 满意度：评价满意度（可选）

- **报表输出**：
  - 每日/每周/每月SLA报表
  - 按人员/班组/类型统计
  - 超时工单清单与原因分析

## 6. 资产台账与生命周期

### 资产分类
- **建筑资产**：楼宇、车位、会议室、仓库
- **设备资产**：电梯、空调、消防、门禁、监控
- **办公资产**：桌椅、电脑、打印机、投影仪
- **能源资产**：变压器、配电柜、水表、电表、燃气表

### 资产信息
```json

  "asset_type": "elevator",

  "location": "1号楼",
  "purchase_date": "2020-01-01",
```

### 生命周期管理
```
采购 → 入库 → 安装 → 运行 → 维保 → 维修 → 报废 → 处置
```

- **维保计划**：
  - 自动生成维保工单
  - 到期前提醒（提前7/3/1天）
  - 维保记录与资产关联
  - 维保成本统计

- **维修记录**：
  - 故障类型、故障原因
  - 维修时长、维修成本
  - 更换配件记录
  - 维修人员

- **故障分析**：
  - 故障频率统计
  - 故障根因分析
  - 高故障设备清单
  - 建议更换/改造

## 7. 能耗分析与优化

### 能耗监测
- **实时监测**：
  - 总能耗：水/电/气/热
  - 分区能耗：按楼宇/楼层/租户
  - 分类能耗：照明/空调/电梯/设备

- **环比同比**：
  - 本月 vs 上月
  - 本年 vs 去年
  - 工作日 vs 节假日

### 异常告警
- **阈值告警**：
  - 单位时间能耗超过阈值
  - 同比增长超过X%
  - 空置区域能耗异常

- **趋势告警**：
  - 能耗持续上升
  - 能耗波动异常
  - 峰谷差异过大

### 能耗分析
- **对标分析**：
  - 行业标杆对比
  - 相似园区对比
  - 历史最优对比

- **碳排放**：
  - 根据能耗计算碳排放
  - 碳中和路径规划
  - 节能减排效果评估

### 节能优化
- **照明优化**：
  - 根据人员在场自动开关灯
  - 根据自然光调节亮度
  - 定时开关策略

- **空调优化**：
  - 根据人员密度调节温度
  - 根据天气预报提前调整
  - 分区控制避免浪费

- **设备优化**：
  - 错峰用电降低成本
  - 老旧设备改造建议
  - 能效对比与替换方案

## 8. 与机器人/AI的融合

### 机器人巡检接入
- **机器人作为"移动传感器"**：
  - 定期巡检任务自动下发
  - 巡检路线与点位配置
  - 巡检数据自动回传

- **巡检内容**：
  - 视频巡检（环境、设备、安全）
  - 传感器巡检（温湿度、气体、噪音）
  - 异常事件自动上报（烟火、积水、异常声音）

- **数据联动**：
  - 巡检发现异常自动创建工单
  - 与人工巡检数据对比验证
  - 形成巡检报告与趋势分析

### AI视频分析
- **安防场景**：
  - 人员闯入、攀爬、聚集
  - 烟火检测、打架斗殴
  - 消防通道堵塞

- **运营场景**：
  - 人流统计与热力分析
  - 车辆违停检测
  - 垃圾堆放检测

- **能耗场景**：
  - 空调/照明开启无人使用
  - 水龙头未关检测
  - 设备运行异常检测

### AI辅助决策（可选）
- **事件分类**：自动识别事件类型并打标签
- **处置建议**：基于历史数据推荐处置方案
- **根因分析**：关联历史事件分析根本原因
- **趋势预测**：预测设备故障、能耗趋势

## 10. 试点切入建议

### 方案一：事件中心+工单闭环（推荐）
**优势**：最容易见效，投入少，周期短

- **第1周**：
  - 部署事件中心平台
  - 接入2-3类告警源（监控/设备/人工报修）
  - 配置工单流程与人员

- **第2-4周**：
  - 移动端APP上线
  - 全员培训与试运行
  - 收集反馈优化流程

- **第2个月**：
  - 接入更多系统（门禁/停车/能耗）
  - 部署指挥大屏
  - SLA报表上线

**验收指标**：
- 工单响应时间<30分钟
- 工单处理及时率>90%
- 用户满意度>85%

### 方案二：能耗分析先行
**优势**：降本增效看得见，领导重视

- **第1周**：
  - 部署能耗采集设备（电表/水表/传感器）
  - 配置能耗监测平台

- **第2-4周**：
  - 能耗数据积累与分析
  - 识别能耗异常点
  - 制定节能方案

- **第2个月**：
  - 实施节能措施
  - 对比节能效果
  - 扩展到更多区域

**验收指标**：
- 能耗降低10%
- 成本节约X万元/年
- 碳排放减少Y吨/年

### 方案三：安防联动先行
**优势**：安全合规必须做，容易拿预算

- **第1周**：
  - 视频监控接入AI分析平台
  - 配置告警规则

- **第2-4周**：
  - 告警与工单联动
  - 移动端处置闭环
  - 历史事件回溯

- **第2个月**：
  - 接入门禁/停车/访客
  - 形成安防闭环
  - 合规报表输出

**验收指标**：
- 安全事件响应时间<5分钟
- 误报率<10%
- 满足等保合规要求

## 12. 差异化竞争力

### 我们的优势
1. **不依赖硬件**：
   - 软件平台为主，兼容主流设备
   - 避免硬件绑定，降低成本
   - 快速部署，2周可用

2. **事件驱动架构**：
   - 以事件为核心串联业务
   - 处置闭环可追溯
   - 数据沉淀形成运营资产

3. **可配置化**：
   - 工单流程自定义
   - 告警规则灵活配置
   - 适配不同园区场景

4. **移动端优先**：
   - 小程序/APP体验优秀
   - 扫码、拍照、语音录入
   - 处置人员随时随地操作

5. **快速见效**：
   - 从事件中心切入，4周见效
   - 分阶段扩展，投入可控
   - 每个阶段都有明确价值
