# 企业智能体平台（Agent Platform）

> 可控、可追溯、可集成的 AI 工作流引擎
> 适用：售前获客 / 运维自动化 / 知识问答 / 业务助手

---

## 1. 为什么不是"聊天机器人"

### 聊天机器人 vs 智能体
| 维度 | 聊天机器人 | 智能体（Agent） |
|------|-----------|----------------|
| 定位 | 回答问题 | 自动完成任务 |
| 能力 | 检索+生成 | 规划+执行+验证 |
| 工具 | 无或简单调用 | 丰富工具编排 |
| 结果 | 文本回复 | 可操作结果（如生成文档/调用API） |
| 可控性 | 难以保证准确性 | 结果可验证/可追溯 |

### 真需求
- **自动完成任务**，而不是只回答问题
- **生成可交付物**（如方案文档/工作量评估/分析报告）
- **调用业务系统**（如创建工单/查询数据/发送通知）
- **多步骤规划**（如"帮我做竞品分析"需要：搜索→整理→对比→生成报告）

### 企业要求
- **权限管控**：不同用户能调用不同工具
- **审计追溯**：每一步操作有记录
- **流程管控**：关键操作需审批
- **稳定性**：不能因为AI不稳定导致业务中断
- **成本可控**：推理成本、API调用成本可预测

## 2. 平台能力定义

### 多 Profile 切换
- **招商 Profile**：
  - 工具：企查查/天眼查、招商政策库、项目匹配
  - 输出：企业画像、投资意向分析、招商话术

- **园区运营 Profile**：
  - 工具：园区数据查询、工单系统、能耗分析
  - 输出：运营报表、异常分析、优化建议

- **康养服务 Profile**：
  - 工具：健康档案查询、风险评估、处方库
  - 输出：健康建议、用药指导、复诊提醒

- **政企售前 Profile**：
  - 工具：需求澄清问卷、方案模板库、工作量评估模型
  - 输出：需求分析报告、方案大纲、报价单

**切换方式**：
- 用户选择或自动识别（根据提问内容）
- 每个Profile有独立的提示词、工具集、知识库

### 工具编排
- **检索工具**：
  - 向量检索（RAG）
  - 全文检索
  - SQL查询
  - API调用

- **写文档工具**：
  - Word/PDF生成
  - PPT生成（基于模板）
  - Excel报表生成

- **生成报表工具**：
  - 数据查询+图表生成
  - 对比分析（同比/环比）
  - 趋势预测

- **业务API工具**：
  - 创建工单
  - 发送通知
  - 查询库存/订单/客户
  - 调用第三方服务（如地图/天气/物流）

**编排方式**：
- 顺序执行（Step1→Step2→Step3）
- 并行执行（同时调用多个工具）
- 条件分支（如果...则...）
- 循环执行（直到满足条件）

### 结果可追溯
- **执行链路**：
  - 输入：用户问题
  - 规划：拆解为N个步骤
  - 执行：每步的工具调用、参数、结果
  - 输出：最终回复

- **证据链**：
  - 每个结论有来源（如"根据XX文档第3页"）
  - 数据有查询SQL
  - 生成内容有模板与填充数据

- **可回溯**：
  - 用户可以查看完整执行过程
  - 出错可以定位具体哪一步
  - 可以重新执行某一步

## 4. 核心闭环：Ask → Plan → Act → Verify → Report

### Ask：需求澄清（问对问题）
**问题**：用户需求往往模糊
- "帮我做竞品分析" → 分析哪些竞品？哪些维度？
- "生成月报" → 哪个月？包含哪些指标？

**解决**：
- 预设澄清问题（如选择题/填空题）
- 智能提问（根据上下文自动补充）
- 用户确认后再执行

**示例**：
```
用户："帮我做竞品分析"
Agent："好的，请问：
  1. 需要分析哪些竞品？（可输入公司名或让我推荐）
  2. 分析哪些维度？（产品功能/价格/市场份额/技术栈/客户评价）
  3. 输出格式？（PPT/Word/PDF）"
用户："阿里云、腾讯云，分析产品功能和价格，输出Word"
Agent："收到，开始执行..."
```

### Plan：规划步骤（工具选择与参数）
**任务**：竞品分析（阿里云、腾讯云，产品功能+价格，Word）

**规划**：
```
Step 1: 搜索阿里云产品信息（工具：Web搜索）
Step 2: 搜索腾讯云产品信息（工具：Web搜索）
Step 3: 提取产品功能列表（工具：信息提取）
Step 4: 提取价格信息（工具：信息提取+表格化）
Step 5: 对比分析（工具：对比模板）
Step 6: 生成Word文档（工具：Word生成器）
```

**优化**：
- Step 1和Step 2可以并行执行
- Step 3和Step 4可以合并（同时提取功能和价格）

### Act：执行调用（可插拔）
**工具调用**：
```python
# Step 1: Web搜索
result_1 = web_search(
    query="阿里云产品功能和价格 2026",
    max_results=10
)

# Step 2: Web搜索
result_2 = web_search(
    query="腾讯云产品功能和价格 2026",
    max_results=10
)

# Step 3: 信息提取
alicloud_info = extract_info(
    content=result_1,
    schema={"products": [...], "pricing": [...]}
)

tencentcloud_info = extract_info(
    content=result_2,
    schema={"products": [...], "pricing": [...]}
)

# Step 4: 对比分析
comparison = compare(
    data_a=alicloud_info,
    data_b=tencentcloud_info,
    dimensions=["products", "pricing"]
)

# Step 5: 生成Word
doc = generate_word(
    template="竞品分析模板.docx",
    data=comparison
)
```

### Verify：验证与回滚（失败策略）
**验证点**：
- 工具返回结果是否为空？
- 数据是否完整（如价格信息缺失）？
- 结果是否符合预期格式？

**失败策略**：
- **重试**：工具调用失败，自动重试3次
- **降级**：主要数据源失败，切换备用数据源
- **跳过**：非关键步骤失败，跳过并标记
- **人工介入**：关键步骤失败，通知用户

**示例**：
```
Step 2失败（腾讯云搜索无结果）
→ 重试2次仍失败
→ 降级：使用缓存数据（上次搜索结果）
→ 标记：数据可能不是最新
→ 继续执行
```

### Report：报告生成（对人可读）
**报告内容**：
- **执行摘要**：完成了什么任务，用了多长时间
- **结果展示**：核心结论、图表、数据
- **数据来源**：引用了哪些资料（带链接）
- **置信度**：结果的可信度评估
- **建议行动**：下一步建议

**示例**：
```markdown
# 阿里云 vs 腾讯云竞品分析报告

## 执行摘要
- 分析维度：产品功能、价格
- 数据来源：官网、第三方评测（2026-02-09）
- 执行时长：2分钟

## 产品功能对比
| 功能 | 阿里云 | 腾讯云 |
|------|--------|--------|

| 存储 | OSS/对象存储 | COS/对象存储 |
| 数据库 | RDS/PolarDB | TencentDB/TDSQL |
| AI | 通义千问/PAI | 混元/TI |

（表格+图表）

- 腾讯云在社交生态集成上更强
- 建议：根据业务场景选择（如有企业微信推荐腾讯云）

## 数据来源

- 腾讯云官网：https://...
- 评测文章：https://...

置信度：85%（价格数据可能有波动）
```

## 6. 典型场景 2：运维/安全自动化

### 场景描述
安全告警触发，需要：
1. 自动关联分析（是否攻击链）
2. 生成事件摘要
3. 给出处置建议
4. 自动创建工单（或执行自动化脚本）

### 工作流
```
输入：告警ID

Step 1: 获取告警详情
- 工具：告警系统API
- 输出：告警类型、来源IP、目标资产、时间等

Step 2: 关联分析
- 工具：安全日志查询（SQL）
- 输出：同一IP/用户/资产的其他事件（时间窗内）

Step 3: 威胁情报查询
- 工具：威胁情报API
- 输出：该IP是否在黑名单、历史恶意行为

Step 4: 攻击链识别
- 工具：规则引擎+LLM分析
- 输出：是否形成攻击链（如扫描→漏洞利用→权限提升）

Step 5: 生成摘要与建议
- 工具：LLM总结
- 输出：
  - 事件摘要（谁在何时对哪个资产做了什么）
  - 风险评估（高/中/低）
  - 处置建议（封禁IP/隔离主机/人工复核）

Step 6: 执行处置（可选）
- 工具：自动化脚本/SOAR
- 输出：执行结果（如已封禁IP）

Step 7: 创建工单
- 工具：工单系统API
- 输出：工单ID，分配给安全团队
```

## 7. 成本与稳定性（核心优势点）

### 本地模型优先
**策略**：
- **日常任务**：使用本地模型（Ollama/私有化模型）
  - 如需求澄清、摘要生成、简单分析
  - 成本：几乎为0（只有电费+服务器）

- **复杂任务**：使用云模型（可选）
  - 如复杂推理、创意生成
  - 成本：按需付费，可控

**对比**：
| 方案 | 成本 | 稳定性 | 数据安全 |
|------|------|--------|---------|
| 纯云模型 | 高（每次调用计费） | 依赖网络+厂商 | 数据出网 |
| 纯本地模型 | 低（一次性部署） | 高（不依赖外网） | 数据不出网 |
| 混合（我们） | 可控（大部分本地） | 高 | 敏感数据不出网 |

### 热点缓存
**策略**：
- **问题缓存**：相同问题直接返回缓存结果
  - 如"最新价格是多少？"今天已问过10次，后续直接返回
  - 节省推理成本

- **工具结果缓存**：
  - 如Web搜索结果缓存1小时
  - 数据库查询结果缓存5分钟
  - 避免重复调用

**效果**：
- 成本降低60%（热点问题不重复推理）
- 响应速度提升80%（缓存命中1秒返回）

### 质量门禁
**策略**：不达标不输出"结论"

- **完整性检查**：
  - 必要字段是否齐全？
  - 数据来源是否标注？

- **准确性检查**：
  - 数值是否在合理范围？
  - 逻辑是否自洽？

- **置信度评估**：
  - 数据来源权威性
  - 推理过程可靠性
  - 低于阈值（如60%）不输出或标记"待人工确认"

**示例**：
```
任务：生成工作量评估
检查：
  ✅ 项目名称: XX园区智慧化
  ✅ 工作内容: 事件中心+工单系统
  ❌ 人天估算: 缺失（找不到相似案例）

结果：不输出完整报告，改为：
  "抱歉，由于缺少相似案例，无法给出准确的工作量估算。
   建议：人工评估 或 提供更多项目细节后重试。"
```
