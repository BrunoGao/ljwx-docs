## 技术栈

### 默认技术栈

| 类别 | 技术 | 说明 |
|------|------|------|
| 消息队列 | Apache Kafka | 高吞吐、持久化 |
| 数据库 | PostgreSQL + TimescaleDB | 时序数据 + 复杂查询 |
| 缓存 | Redis | 画像、热查询 |
| 调度 | Apache Airflow | 批处理调度 |
| 监控 | Prometheus + Grafana | 指标采集 + 可视化 |

### 可选增强

| 组件 | 用途 | 何时启用 |
|------|------|---------|
| Apache Flink | 实时计算 | 延迟 < 3秒 |
| ClickHouse | OLAP 分析 | QPS > 50 |
| Neo4j | 图关联分析 | 需要关联分析 |
| MLflow | 模型管理 | 模型 > 3个 |

## 核心能力

### 1. 可信数据资产
- ✅ 质量可量化（六维质量）
- ✅ 来源可追溯（数据血缘）
- ✅ 变更可审计（操作日志）

### 2. 稳定数据处理
- ✅ 不丢（断网续传 + 重试）
- ✅ 不重（幂等 + 去重）
- ✅ 可恢复（回放 + 重算）

### 3. 可解释风险预警
- ✅ 低误报（< 5%）
- ✅ 可解释（规则 + 证据）
- ✅ 可优化（闭环反馈）

### 4. 长期运行保障
- ✅ 可观测（监控 + 追踪）
- ✅ 可应急（预案 + 演练）
- ✅ 可扩展（模块化 + 增强）
