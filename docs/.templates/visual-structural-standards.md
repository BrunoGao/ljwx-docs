## 🎯 设计理念

### 核心原则

1. **每屏回答一个问题** - 避免信息过载，清晰的视觉层次
2. **垂直递进展示流程** - 方法论文章使用垂直布局展示逻辑流程
3. **专业白皮书风格** - 更像"论文入口"而非"博客文章"
4. **明确的主锚点** - 每个区域有清晰的视觉重心，避免元素竞争

### 定位

- ❌ **不是**：技术教程、操作手册、博客文章
- ✅ **是**：方法论白皮书、治理模型文档、决策参考

## 🏗️ 信息架构标准

### 四区块结构（A/B/C/D）

所有方法论文章的"写在前面"部分应采用以下四区块结构：

```
┌─────────────────────────────────────┐
│  A区：方法论声明（必读）              │  ← 灰色主题，声明性质
├─────────────────────────────────────┤
│  B区：核心价值主线                    │  ← 垂直递进，编号圆圈
│    1. Why（决策逻辑）                 │     蓝色 #3451b2
│    ↓                                 │
│    2. What（方法论定义）              │     绿色 #10b981
│    ↓                                 │
│    3. How（实施路径）                 │     橙色 #f59e0b
│    ↓                                 │
│    4. Lessons（经验教训）             │     红色 #ef4444
├─────────────────────────────────────┤
│  C区：目标受众                        │  ← 2列网格，适合/不适合
├─────────────────────────────────────┤
│  D区：阅读导航                        │  ← 路径选择器风格
└─────────────────────────────────────┘
```

### 区块详细说明

#### A区：方法论声明

**目的**：明确文章定位，过滤读者

**视觉特征**：
- 灰色主题（#6b7280）
- 左侧 3px 边框
- 低饱和度渐变背景（0.03 → 0.01）
- 圆角 8px

**内容要素**：
- 文章类型声明（方法论 vs 教程）
- 核心价值三要素
- 适用/不适用场景

#### B区：核心价值主线

**目的**：展示方法论的逻辑流程

**视觉特征**：
- 垂直递进布局（单列）
- 编号圆圈（32px，绝对定位）
- 连接箭头（↓，灰色 #9ca3af）
- 四色系统：蓝→绿→橙→红
- 渐变饱和度：0.04 → 0.01

**内容要素**：
1. Why - 决策逻辑（为什么需要）
2. What - 方法论定义（是什么）
3. How - 实施路径（怎么做）
4. Lessons - 经验教训（避免什么）

#### C区：目标受众

**目的**：帮助读者快速判断是否适合阅读

**视觉特征**：
- 2列网格布局
- 左列：绿色（适合）#10b981
- 右列：红色（不适合）#ef4444
- 圆角 12px
- 边框 1px

**内容要素**：
- 4个适合场景
- 4个不适合场景

#### D区：阅读导航

**目的**：提供个性化阅读路径

**视觉特征**：
- 蓝色主题（#3451b2）
- 编号场景（①②③④）
- 箭头指向（→）
- 链接到具体章节

**内容要素**：
- 4个典型阅读场景
- 每个场景推荐的章节组合

## 🎨 视觉设计规范

### 色彩系统

#### 主色调

| 颜色 | 色值 | 用途 | 渐变饱和度 |
|------|------|------|-----------|
| 蓝色 | #3451b2 | Why、品牌色、链接 | 0.04 → 0.01 |
| 绿色 | #10b981 | What、成功、适合 | 0.04 → 0.01 |
| 橙色 | #f59e0b | How、警告、注意 | 0.04 → 0.01 |
| 红色 | #ef4444 | Lessons、危险、不适合 | 0.04 → 0.01 |
| 灰色 | #6b7280 | 声明、中性 | 0.03 → 0.01 |

#### 渐变公式

```css
background: linear-gradient(135deg,
  rgba(R, G, B, 0.04) 0%,
  rgba(R, G, B, 0.01) 100%);
```

**重要**：饱和度从 0.05 降低到 0.04，更专业、更克制

### 圆角规范

| 元素 | 圆角值 | 说明 |
|------|--------|------|
| 大容器 | 12px | C区、D区 |
| 中容器 | 8px | A区、B区单元 |
| 小元素 | 50% | 编号圆圈 |

### 间距规范

| 类型 | 间距值 | 说明 |
|------|--------|------|
| 区块间距 | 32px | A/B/C/D 区块之间 |
| 内容间距 | 24px | 区块内部 padding |
| 元素间距 | 16px | 小元素之间 |
| 箭头间距 | 16px | 垂直递进箭头 |

### 边框规范

| 类型 | 样式 | 说明 |
|------|------|------|
| 左侧强调 | 3px solid | A区、B区单元 |
| 容器边框 | 1px solid | C区、D区 |
| 透明度 | rgba(color, 0.2) | 边框颜色 |

## 🧩 组件库

### 1. 方法论声明容器

```html
<div style="padding: 24px; border-left: 3px solid #6b7280; background: linear-gradient(135deg, rgba(107, 114, 128, 0.03) 0%, rgba(107, 114, 128, 0.01) 100%); border-radius: 8px; margin: 32px 0;">

### 方法论声明（必读）

这是一篇 **[方法论类型]** 文章，而不是技术教程。

[内容...]

</div>
```

### 2. 垂直递进单元（带编号圆圈）

```html
<div style="position: relative; margin-bottom: 32px;">
  <div style="position: absolute; left: 0; top: 8px; width: 32px; height: 32px; background: #3451b2; border-radius: 50%; display: flex; align-items: center; justify-content: center; color: white; font-weight: 600; font-size: 14px;">1</div>
  <div style="padding: 20px 24px; border-left: 3px solid #3451b2; background: linear-gradient(135deg, rgba(52, 81, 178, 0.04) 0%, rgba(52, 81, 178, 0.01) 100%); border-radius: 0 8px 8px 0; margin-left: 16px;">
    <div style="font-size: 20px; font-weight: 600; color: #3451b2; margin-bottom: 8px;">[标题]</div>
    <div style="color: var(--vp-c-text-2); line-height: 1.7;">[描述]</div>
  </div>
</div>
```

**颜色变体**：
- 编号1（Why）：#3451b2（蓝色）
- 编号2（What）：#10b981（绿色）
- 编号3（How）：#f59e0b（橙色）
- 编号4（Lessons）：#ef4444（红色）

### 3. 连接箭头

```html
<div style="margin-left: 16px; margin-bottom: 16px; color: #9ca3af; font-size: 20px;">↓</div>
```

### 4. 目标受众卡片（2列网格）

```html
<div style="display: grid; grid-template-columns: 1fr 1fr; gap: 24px; margin: 32px 0; max-width: 900px;">

<!-- 适合阅读 -->
<div style="padding: 24px; background: linear-gradient(135deg, rgba(16, 185, 129, 0.05) 0%, rgba(16, 185, 129, 0.02) 100%); border-radius: 12px; border: 1px solid rgba(16, 185, 129, 0.2);">
  <div style="font-size: 18px; font-weight: 600; color: #10b981; margin-bottom: 16px;">✓ 适合阅读</div>
  <div style="color: var(--vp-c-text-2); line-height: 1.8;">
    • [场景1]<br/>
    • [场景2]<br/>
    • [场景3]<br/>
    • [场景4]
  </div>
</div>

<!-- 不适合阅读 -->
<div style="padding: 24px; background: linear-gradient(135deg, rgba(239, 68, 68, 0.05) 0%, rgba(239, 68, 68, 0.02) 100%); border-radius: 12px; border: 1px solid rgba(239, 68, 68, 0.2);">
  <div style="font-size: 18px; font-weight: 600; color: #ef4444; margin-bottom: 16px;">✗ 不适合阅读</div>
  <div style="color: var(--vp-c-text-2); line-height: 1.8;">
    • [场景1]<br/>
    • [场景2]<br/>
    • [场景3]<br/>
    • [场景4]
  </div>
</div>

</div>
```

### 5. 阅读导航容器

```html
<div style="padding: 24px; background: linear-gradient(135deg, rgba(52, 81, 178, 0.03) 0%, rgba(52, 81, 178, 0.01) 100%); border-radius: 12px; margin: 32px 0;">

<div style="margin-bottom: 20px;">
  <div style="font-weight: 600; color: var(--vp-c-text-1); margin-bottom: 8px;">① [场景标题]</div>
  <div style="color: var(--vp-c-text-2); padding-left: 20px;">
    → 建议阅读：<a href="#section1">章节1</a> & <a href="#section2">章节2</a>
  </div>
</div>

<!-- 重复其他场景 -->

</div>
```

## 📐 排版规范

### 字体大小

| 元素 | 字号 | 字重 | 说明 |
|------|------|------|------|
| 区块标题 | 20px | 600 | B区单元标题 |
| 卡片标题 | 18px | 600 | C区卡片标题 |
| 正文 | 16px | 400 | 默认正文 |
| 描述文字 | 14px | 400 | 辅助说明 |
| 编号圆圈 | 14px | 600 | 圆圈内数字 |

### 行高

| 元素 | 行高 | 说明 |
|------|------|------|
| 标题 | 1.4 | 紧凑 |
| 正文 | 1.7 | 舒适阅读 |
| 列表 | 1.8 | 更宽松 |

### 对齐方式

- **标题**：左对齐
- **正文**：左对齐
- **编号圆圈**：居中对齐（flex）
- **箭头**：左对齐（margin-left: 16px）

## ✍️ 内容写作指南

### A区：方法论声明

**写作要点**：
1. 第一句明确文章类型（方法论 vs 教程）
2. 第二句说明核心价值（三要素）
3. 明确适用/不适用场景

**示例**：
```
这是一篇 **交付方法论** 文章，而不是技术教程。

它不讲如何部署 Argo CD，而是总结我们在多个 **私有化项目** 中实践 GitOps 后，形成的一套「**可审计、可回滚、可规模化**」的交付治理模型。

**适用场景**：私有化部署、多环境管理、需要审计追溯

**不适用场景**：单体应用、简单脚本部署、无合规要求
```

### B区：核心价值主线

**写作要点**：
1. **Why**：用问题驱动，说明痛点
2. **What**：给出清晰定义和核心原则
3. **How**：概述实施路径（不是详细步骤）
4. **Lessons**：总结反模式和教训

**字数控制**：每个单元 1-2 句话，简洁有力

### C区：目标受众

**写作要点**：
1. 每个场景用一句话描述
2. 适合/不适合各4个场景
3. 场景要具体、可识别

**示例**：
```
✓ 适合阅读
• 正在做私有化 / 内网交付
• 多环境、多项目，变更频繁
• 有审计、合规、回滚要求
• 团队协作成本高、责任边界不清

✗ 不适合阅读
• 单体应用，手动部署即可
• 云原生 SaaS，无私有化需求
• 只想学 Argo CD 操作
• 团队 < 3 人，项目 < 2 个
```

### D区：阅读导航

**写作要点**：
1. 4个典型读者场景
2. 每个场景推荐 2-3 个章节
3. 使用编号（①②③④）和箭头（→）

**示例**：
```
① 正在评估 GitOps？
→ 建议阅读：背景 & 为什么不是传统 CI/CD

② 已决定采用，想了解实施路径？
→ 建议阅读：方法论定义 & 实施步骤

③ 已在实践，想避免踩坑？
→ 建议阅读：反模式 & 最佳实践

④ 想全面了解？
→ 建议阅读：完整阅读
```

## 📊 表格规范

### 对比表格

用于方法论对比、效果对比等：

```markdown
| 维度 | 方案A | 方案B | 说明 |
|------|-------|-------|------|
| [维度1] | [描述] | [描述] | [说明] |
| [维度2] | [描述] | [描述] | [说明] |
```

**要点**：
- 第一列：对比维度
- 中间列：具体方案
- 最后一列：补充说明（可选）

### 数据表格

用于展示效果数据：

```markdown
| 指标 | 实施前 | 实施后 | 提升幅度 |
|------|--------|--------|----------|
| [指标1] | [数据] | [数据] | [百分比] |
| [指标2] | [数据] | [数据] | [百分比] |
```

**要点**：
- 使用具体数据，不要模糊描述
- 提升幅度用百分比或倍数

## 🎯 检查清单

在发布文章前，请检查以下项目：

### 信息架构
- [ ] 采用 A/B/C/D 四区块结构
- [ ] A区有明确的方法论声明
- [ ] B区使用垂直递进布局（不是网格）
- [ ] C区有适合/不适合两列
- [ ] D区有4个阅读路径

### 视觉设计
- [ ] 编号圆圈使用四色系统（蓝绿橙红）
- [ ] 渐变饱和度为 0.04 → 0.01
- [ ] 圆角规范：12px（大）/ 8px（中）/ 50%（圆）
- [ ] 间距规范：32px（区块）/ 24px（内容）/ 16px（元素）
- [ ] 使用连接箭头（↓）展示流程

### 内容质量
- [ ] 方法论声明清晰（不是教程）
- [ ] 核心价值主线简洁（每单元1-2句）
- [ ] 目标受众具体可识别
- [ ] 阅读导航有明确推荐

### 排版细节
- [ ] 标题字号 20px / 18px
- [ ] 正文行高 1.7 / 1.8
- [ ] 表格数据具体（不模糊）
- [ ] 链接颜色使用品牌色
