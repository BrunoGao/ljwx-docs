## 对外宣讲版 v4.0（政府采购/业主方版）

### P1 封面

# 智慧园区与智慧楼宇一体化平台

**事件驱动的园区运营 OS：把告警、工单、巡检、能耗联成可验收闭环，让处置从"靠电话"变成"靠系统"。**

适用：产业园区 / 办公楼宇 / 工业园区 / 政府园区

> 本项目为纯软件平台 + 接口集成：以现网子系统接口开放为前提，最大化复用存量设备与系统，不替换、不绑定硬件品牌。

### P2 一页读懂核心价值

**一个事件中心，把告警、工单、巡检、能耗拉成可验收闭环。**

三张牌（可验收口径）：

- **快：响应可压到分钟级（可验收）**——典型试点场景，派发→接单 ≤3–5 分钟（取决于值班制度与触达通道配置）；高频轻量事件全流程闭环可达 15–30 分钟
- **稳：4 周 MVP 上线（可控）**——只做"2–3 类事件源接入 + 工单闭环 + 移动端"，其余严格分期，每期有独立里程碑与验收标准
- **省：不绑硬件（可复用）**——以标准协议/API 接入为主，最大化复用存量系统与设备，显著降低初期投入

前置条件（必须项）：各子系统接口清单与测试账号、内网部署资源与网络策略、设备点位表、组织架构与班组信息齐备。

> 闭环时长按场景分档：高频轻量场景（通道占用、违停、一般设备告警）可达 15–30 分钟；涉及跨专业协调、备件调配、停机审批的事件按更长 SLA 分档计时。

### P3 现状痛点：系统多、数据乱、处置慢

**一个典型园区有 6–10 套独立系统，但处置一个问题要打 5 个电话。**

```
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│ 门禁 │ │ 停车 │ │ 监控 │ │ 能耗 │ │ 工单 │ │ 访客 │
│ 系统 │ │ 系统 │ │ 系统 │ │ 系统 │ │ 系统 │ │ 系统 │
└──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘
   ✗ 不共享   ✗ 不联动   ✗ 不闭环   ✗ 口径不一  ✗ 无追溯   ✗ 不打通
```

**系统多**：门禁、停车、监控、能耗、工单、访客各自为政，来自不同厂商、不同年份采购

**数据乱**：门禁说在线 1000 人，能耗推算 800 人；安防发现问题，工单系统不知道；设备坏了，维保记录散落在纸质台账和 Excel 里

**处置慢**：发现 → 电话报告 → 层层转达 → 手动派单 → 处理 → 无人验收

```
传统处置时间轴：
发现问题 ──30min──▶ 电话上报 ──60min──▶ 派单 ──数小时──▶ 处理 ──?──▶ 验收缺失
            ▲                                                        ▲
         依赖人工发现                                            无法追溯闭环
```

### P4 我们的解法：统一事件中心串联一切

**所有告警、报修、巡检统一入口 → 自动派单 → 闭环验收 → 数据沉淀。**

```
事件来源              事件中心                处置闭环              数据价值
┌─────────┐      ┌──────────────┐      ┌──────────────┐      ┌─────────────┐
│ 安防告警 │─┐    │              │      │              │      │             │
│ 设备故障 │─┤    │  自动分级    │      │  移动端接单  │      │  统计分析   │
│ 能耗异常 │─┼───▶│  智能派单    │─────▶│  现场处理    │─────▶│  根因分析   │
│ 人工报修 │─┤    │  SLA 计时     │      │  证据上传    │      │  趋势预测   │
│ 巡检发现 │─┘    │  超时升级    │      │  验收归档    │      │  持续优化   │
└─────────┘      └──────────────┘      └──────────────┘      └─────────────┘
```

**三个统一**：统一数据底座（消除口径矛盾）、统一事件中心（处置全链路）、统一运营看板（一屏掌控）

**以事件为主线**：不是"功能堆砌再拉通"，而是事件天然串联——告警产生事件，事件生成工单，工单驱动处置，处置沉淀数据

### P5 三个可记忆的差异化

**不绑硬件 · 事件驱动 · 可配置——三个词说清和其他方案的区别。**

| 对比维度 | 设备厂商主导型 | 总包平台型 | 本平台（软件中台型） |
|---------|-------------|----------|-------------------|
| 硬件依赖 | 强绑定自有设备 | 倾向推荐自有设备 | 品牌无关，复用存量 |
| 交付周期 | 3–6 个月 | 6–12 个月 | MVP 4 周，全量 12 周 |
| 业务联动 | 自有设备间好，跨品牌弱 | 功能全但联动需定制 | 事件驱动天然联动 |
| 灵活性 | 方案相对固定 | 需专业实施团队 | 流程/规则/SLA 可视化配置 |
| 信创适配 | 视厂商情况 | 部分支持 | 支持信创环境部署（见 P12） |

**不绑硬件**：标准协议接入 + 插件化适配器，保护存量投入

**事件驱动**：从事件闭环出发倒推接入需求，天然打通各子系统

**可配置**：工单流程、派单规则、告警阈值、SLA 标准全部可视化配置，适配不同管理模式

> 纯软件 + 接口集成边界说明：如遇封闭/私有协议的存量子系统，需原厂配合开放接口或通过网关/中间库方式接入，相关工作量在 PoC 阶段评估确认。

### P6 闭环怎么来的：SLA 三段计时 + 分段升级

**接单、到场、验收分段计时——每段可配置、可验收、可审计。**

```
触发方式：AI 视频告警 / 传感器告警 / 人工巡检 / 租户扫码报修（任一触发）
    │
    ▼  ≤3 秒（自动）
  事件中心自动分级 → 匹配工种 → 就近派单
    │
    ▼  接单时长计时开始
  处置人员手机收到推送 + 短信 → 查看详情 + 一键导航
    │
    ▼  到场时长计时开始
  到达现场 → 处置 → 拍照/视频上传处理结果
    │
    ▼  验收时长计时开始
  验收人移动端确认 → 通过则归档 / 不通过则回退重处理
    │
    ▼
  事件关闭 → 数据沉淀 → 纳入统计报表
```

**SLA 三段计时（均可配置、可验收）**：接单时长（派发→接单）、到场时长（接单→到达现场）、验收时长（处理完成→验收通过）

**分段超时升级**：未接单超时 → 升级至班组长/主管；未到场超时 → 升级并启动备岗；未验收超时 → 升级至管理员。支持电话 + 短信+APP 多通道触达，升级规则按事件等级和甲方管理制度配置

**"15–30 分钟"限定高频轻量场景**：通道占用、车辆违停、轻微设备告警、一般报修等；复杂维修、跨专业协调、需备件/停机审批的事件按 P1/P2 分档设定更长 SLA

### P8 统一事件中心：平台的核心枢纽

**所有事件统一入口、统一分级、统一派单、统一闭环——全链路可追溯可审计。**

**事件模型核心字段**：事件类型（告警/故障/报修/巡检）、严重等级（P0–P3）、位置（楼宇/楼层/区域）、关联资产、当前状态、责任人

**试点首批接入两类事件源**：

安防告警——AI 视频分析/传感器触发的入侵、聚集、烟火、消防通道堵塞等事件，自动进入事件中心

设备故障——门禁/电梯/空调/消防等设备的故障、离线、参数异常告警，自动进入事件中心

**自动派发策略**：按工种匹配（电工/水暖/保安/保洁）、按位置就近、按负载均衡；P0 紧急事件电话 + 短信+APP 三通道同时触达

**全链路审计**：每个状态变更记录操作人、操作时间、操作内容，形成不可抵赖的证据链；审计日志可对接甲方集中日志审计平台

**数据沉淀**：闭环后自动统计高发类型、高发区域、高发时段，驱动月度运营报告与预防性改进

### P9 能耗分析与优化 + 移动端体验

**上半页：能耗闭环——发现异常 → 自动工单 → 复盘节能**

**实时监测**：水/电/气/热全量采集，按楼宇、楼层、租户、用途多维统计

**异常告警**：阈值告警（超标/同比异常/空置区域能耗）、趋势告警（持续上升/波动异常），告警自动进入事件中心触发排查工单

**对标分析**：行业标杆/相似园区/自身历史最优，碳排放自动计算

**节能联动**：照明（人走灯灭/自然光调节）、空调（人员密度/天气预报/分区控制）、设备（错峰用电/能效对比）

> 场景实例：某园区接入后第一周发现某楼层夜间电耗异常——空调定时失效持续运行。修复后该区域夜间电耗下降 70%。

---

**下半页：移动端——一线处置人员的主战场**

**扫码报修**：扫设备二维码自动创建工单，关联设备台账与维保记录

**语音录入**：现场语音转文字记录处理过程

**拍照/视频上传**：处置前后照片对比，形成验收证据链

**一键导航**：点击工单直接导航到事件位置

**班组工作台**：待办清单、SLA 倒计时、当日路线

**多端覆盖**：小程序（零安装推广快）+ 原生 APP；管理端支持 Web 和平板

### P10 可选增强：AI 与机器人

**AI 和机器人是让闭环更快、更准的加速器——作为扩展阶段按需引入。**

**AI 视频分析**：烟火/聚集/通道堵塞/违停等检测结果自动进入事件中心

**机器人巡检**：作为移动传感器按路线定时巡检，异常自动上报生成工单

**AI 辅助决策**：事件自动分类打标、基于历史数据推荐处置方案、关联分析根因、预测设备故障趋势

**自然语言交互与报告生成**：管理人员自然语言提问运营数据，周报/月报 AI 撰写初稿

> AI 能力的实际表现受场景复杂度与数据积累量影响，建议通过 PoC 验证效果后在扩展阶段分步上线。

### P11 安全与合规：按等保三级对标 + 信创适配 + 测评配合

**内网部署、数据不出域、信创适配——安全合规不是可选项，是交付基线。**

**数据不出域**：全量内网私有化部署，数据存储与计算均在甲方内网环境，不依赖外部云资源

**身份与权限**：RBAC 细粒度权限、最小必要原则、多租户数据隔离；支持对接甲方统一身份平台（LDAP/AD/SSO，按现网环境适配）

**审计与留存**：全操作审计、关键操作留痕、日志留存策略可按甲方安全基线配置；审计日志可对接甲方集中日志审计平台或集中安全监控平台

**加密与国密**：传输与存储加密能力齐备；如甲方要求适配国密体系，需在部署环境中配套密码服务/国密组件，按项目方案落地

**信创适配**：平台支持在信创环境中运行，包括国产操作系统（麒麟/统信）、国产数据库（达梦/人大金仓/GaussDB）、国产中间件（东方通/宝兰德）；具体适配范围以甲方信创基线为准，在 PoC 阶段完成环境验证

**等保测评配合**：架构与控制项对标等保 2.0 三级要求；可提供控制项覆盖说明、部署拓扑图、账号权限矩阵、日志留存策略等材料；配合测评机构完成测评与整改闭环

> 具体控制项清单与落地方式以甲方安全基线与测评机构要求为准。

### P12 交付与验收：按里程碑交付、按口径验收、按闭环移交

**每个阶段有交付物清单、验收指标和责任边界——可审计、可追溯。**

**里程碑**

```
现场勘察     接口联调     MVP 上线      试点验收     扩容分期
(1–2 天)  →  (1–2 周)  →  (第 4 周)  →  (第 5–6 周) →  (按期推进)
```

**交付物清单（每期均可落档归档）**：接口联调报告（含数据映射说明）、事件/工单流程配置说明书、SLA 口径定义与取数说明、权限矩阵与账号清单、运维手册、培训记录与签到表

**试点阶段验收指标（口径明确、取数可复现）**

| 指标 | 口径定义 | 试点目标 |
|-----|---------|---------|
| 接单时长 | 从系统派发到处置人员确认接单的时长 | ≤5 分钟（P0/P1 事件） |
| SLA 达标率 | 在约定 SLA 时限内完成闭环的事件占比 | ≥85% |
| 移动端覆盖率 | 一线处置人员安装并激活移动端的比例 | ≥90% |
| 闭环率 | 已创建事件中最终完成验收归档的比例 | ≥95% |
| 审计留痕完整性 | 随机抽取事件，全链路操作记录完整的比例 | 100% |

> 指标取数方式：从平台事件台账与 SLA 报表中导出，验收时甲方可独立核查原始数据。

**边界与责任划分（RACI 矩阵简表）**

| 工作事项 | 我方（软件平台方） | 甲方 | 原系统厂商 |
|---------|-----------------|-----|-----------|
| 平台部署与配置 | 负责（R） | 提供部署资源（S） | — |
| 子系统接口开放 | 联调对接（R） | 协调推动（A） | 提供接口与账号（R） |
| 网络策略与安全域 | 提出需求（C） | 审批并实施（R） | — |
| 工单流程与 SLA 配置 | 配置实施（R） | 确认流程（A） | — |
| 移动端推广与培训 | 培训交付（R） | 组织人员参训（A） | — |
| 设备侧故障处理 | 事件记录与工单流转（R） | 归口管理（A） | 维修处置（R） |
| 信创环境适配与验证 | 适配部署（R） | 提供信创基线（S） | — |

> R=负责执行 A=审批/决策 S=支持配合 C=咨询/告知

**变更管理**：接口变更、流程变更、阈值策略变更均在平台中留痕可回滚；重大变更需甲方审批后生效

### P13 客户案例

**两个真实案例，口径统一：背景 → 切入方式 → 可验证结果。**

**案例一：某华东科技产业园（12 栋楼，80+ 入驻企业）**

背景：7 套独立系统、5 个厂商，60 人物业团队，日均工单 40–50 个，响应慢、投诉多。

切入：事件中心 + 工单闭环 + 移动端，首批接入安防告警、设备故障、人工报修三类事件源。4 周 MVP 上线，第 6 周完成试点验收。

结果（上线 6 个月，按平台 SLA 报表统计）：
- 工单平均接单时长：2.1 小时 → 8 分钟
- SLA 达标率：62% → 93%
- 能耗同比下降约 15%（接入能耗模块后第 3 个月统计）

**案例二：某华南政府行政园区（5 栋楼，常驻约 2000 人）**

背景：安全合规刚性要求，安防与办公系统割裂，内网私有化部署，有信创适配要求。

切入：安防联动先行，首批接入 200 路摄像头 + AI 分析，告警与工单联动。部署于甲方内网信创环境（麒麟操作系统 + 达梦数据库）。

结果（上线 4 个月，按事件台账统计）：
