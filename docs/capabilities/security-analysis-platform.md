# 安全数据分析与自动化处置平台

> 日志归一 → 风险识别 → 自动化处置 → 证据链
> 适用：安全运营中心(SOC) / 等保合规 / 安全托管服务

---

## 1. 典型痛点

### 日志多源异构
- **格式混乱**：syslog、JSON、XML、自定义格式
- **字段缺失**：关键信息不全，如时间戳、源IP、用户ID
- **时间不准**：时区不统一、时钟漂移、格式各异
- **编码问题**：UTF-8、GBK混用，特殊字符乱码
- **语义不一**：同一事件不同系统描述方式差异大

### 告警泛滥
- **误报率高**：大量低质量告警，淹没真实威胁
- **关联困难**：单点告警无法形成攻击链全貌
- **处置慢**：人工研判耗时长，MTTR(平均修复时间)高
- **重复告警**：同一问题重复上报，浪费精力
- **缺少优先级**：无法区分紧急与一般告警

### 合规压力
- **等保要求**：等保2.0/3.0安全审计与日志留存
- **行业监管**：金融、医疗、政务等行业专项合规
- **审计溯源**：事后取证困难，证据链不完整
- **数据保留**：海量日志存储与检索性能矛盾
- **权限管控**：审计数据本身的访问权限管理

## 2. 总体方案

### 解决方案架构
```
采集层 → 归一层 → 分析层 → 处置层 → 证据层
```

### 各层职责
- **采集层**：多源日志统一接入（Agent/接口/网关）
- **归一层**：解析、字段映射、标签化、标准化存储
- **分析层**：规则引擎 + 统计分析 + 关联分析 + AI检测
- **处置层**：工单管理 + 自动化脚本 + 封禁联动 + 通知触达
- **证据层**：事件时间线 + 原始日志留存 + 处置记录归档

## 3. 数据模型（安全域）

### 核心实体
```
Asset（资产）: 服务器、网络设备、应用系统、数据库
Identity（身份）: 用户、服务账号、API Key
Event（事件）: 登录、访问、操作、告警
Alert（告警）: 安全告警、合规告警、异常告警
```

### 统一标识
- **traceId**：跨系统事件链路追踪ID
- **eventId**：单条事件全局唯一ID
- **alertId**：告警事件ID
- **caseId**：安全事件案件ID（关联多个告警）

### 可追溯字段（核心）
- **来源信息**：
  - source_system：来源系统名称
  - source_ip：数据源IP
  - collector：采集器标识

- **时间信息**：
  - event_time：事件发生时间（原始）
  - collect_time：采集时间
  - process_time：处理时间
  - alert_time：告警时间

- **处理链路**：
  - process_steps：处理步骤记录（解析→归一→分析→告警）
  - rule_hit：命中的规则列表
  - confidence_score：置信度评分

- **处置信息**：
  - handler：处置人
  - handle_time：处置时间
  - handle_action：处置动作
  - handle_result：处置结果

## 4. 关键能力：关联分析

### 同一主体多事件聚合
- **用户维度**：
  - 同一用户短时间内多次登录失败
  - 同一用户在不同地点同时登录
  - 同一用户异常时段访问敏感资源

- **IP维度**：
  - 同一IP短时间扫描多个端口
  - 同一IP访问多个异常URL
  - 同一IP多次触发WAF规则

- **资产维度**：
  - 同一服务器多个异常进程
  - 同一服务器异常网络连接
  - 同一服务器文件被篡改+异常登录

### 时间窗内异常行为链
```
攻击链示例：
外网扫描 → 漏洞利用 → 权限提升 → 横向移动 → 数据窃取 → 痕迹清除
```

- **时间窗配置**：
  - 短时窗（5-30分钟）：检测快速攻击
  - 中时窗（1-24小时）：检测持续渗透
  - 长时窗（7-30天）：检测APT攻击

- **关联规则**：
  - 时序关联：事件A发生后N分钟内发生事件B
  - 因果关联：事件A是事件B的前置条件
  - 空间关联：同一网段/区域发生的事件

### 置信度评分
```
置信度 = 规则命中权重 × 0.4 + 统计偏离权重 × 0.3 + 历史画像权重 × 0.3
```

- **规则命中**：
  - 高危规则（如SQL注入）：权重1.0
  - 中危规则（如异常登录）：权重0.6
  - 低危规则（如信息泄露）：权重0.3

- **统计偏离**：
  - 与历史基线对比，偏离度越大权重越高
  - 例如：正常访问量100次/小时，突然1000次/小时

- **历史画像**：
  - 该用户/IP/资产历史是否有恶意行为
  - 是否在黑名单或威胁情报库中

### 关联分析输出
- **攻击链路图**：可视化展示攻击全貌
- **受影响资产清单**：哪些资产被波及
- **建议处置措施**：基于场景自动生成
- **相似历史案件**：关联历史相似事件

## 5. 关键能力：自动化处置（SOAR思路）

### 预案编排
```
触发条件 → 自动动作 → 人工审批（可选）→ 执行动作 → 回执确认 → 复盘分析
```

### 预案示例
**预案1：暴力破解自动封禁**
```yaml
name: SSH暴力破解自动封禁
trigger:
  - rule: SSH登录失败次数>10次/5分钟
  - asset_type: Linux服务器
conditions:
  - source_ip not in whitelist
  - failed_count > 10
actions:
  - action: block_ip
    target: firewall
    params:
      ip: ${source_ip}
      duration: 3600  # 封禁1小时
  - action: send_alert
    target: security_team
    params:
      level: P2
      channel: [wechat, email]
approval: not_required
rollback:
  - after: 3600s
    action: unblock_ip
```

**预案2：高危漏洞自动隔离**
```yaml
name: 高危漏洞主机自动隔离
trigger:
  - vuln_level: critical
  - exploit_detected: true
conditions:
  - asset_importance: high
  - exploit_confirmed: true
actions:
  - action: isolate_host
    target: sdn_controller
    params:
      host_ip: ${asset_ip}
      allow_list: [monitor_ip, patch_server_ip]
  - action: create_ticket
    target: itsm
    params:
      title: "高危主机隔离: ${asset_ip}"
      priority: P0
      assign_to: security_ops
approval: required  # 需要安全主管审批
rollback:
  - manual: true  # 手动解除隔离
```

### 动作库（可扩展）
- **网络动作**：
  - 封禁IP/端口（防火墙/WAF）
  - 主机隔离（SDN/网络准入）
  - 会话断开（强制下线）

- **主机动作**：
  - 进程终止（kill恶意进程）
  - 文件隔离/删除
  - 账号禁用/密码重置

- **取证动作**：
  - 内存dump
  - 磁盘镜像
  - 流量抓包
  - 日志快照

- **通知动作**：
  - 微信/钉钉/飞书通知
  - 邮件/短信告警
  - 工单自动创建
  - 值班电话外呼

- **脚本执行**：
  - 执行自定义Shell/Python脚本
  - 调用第三方API
  - 触发Webhook

### 人机协同
- **自动建议**：
  - AI分析生成处置建议
  - 展示预期影响与风险
  - 提供多种处置方案对比

- **人工审批**：
  - 高风险操作需审批（如生产环境封禁）
  - 审批流程可配置（单人/多人/分级）
  - 超时自动升级或自动执行

- **手动接管**：
  - 任何时候可人工中止自动化
  - 人工执行后自动记录操作日志
  - 支持半自动模式（建议+人工确认）

## 6. 合规与审计

### 分级分类
- **数据分级**：
  - 公开级：可公开访问
  - 内部级：内部员工可访问
  - 机密级：需特殊授权
  - 绝密级：最小化权限

- **数据分类**：
  - 用户数据：姓名、手机、身份证等
  - 业务数据：订单、交易、合同等
  - 系统数据：日志、配置、代码等
  - 审计数据：操作记录、访问日志等

### 最小权限原则
- **角色定义**：
  - 安全管理员：全部安全数据访问
  - 安全分析师：只读安全数据，可创建规则
  - 审计员：只读审计数据，不可修改
  - 业务管理员：仅本业务域数据

- **权限审批**：
  - 高权限需审批流程
  - 临时权限自动过期
  - 权限变更全程审计

### 审计报表
- **访问审计**：
  - 谁在何时访问了哪些数据
  - 访问来源（IP/地点）
  - 访问方式（Web/API/SQL）
  - 数据量统计（查询/导出）

- **变更审计**：
  - 配置变更记录
  - 规则变更记录
  - 权限变更记录
  - 系统升级记录

- **处置审计**：
  - 告警处置全流程
  - 自动化执行记录
  - 人工操作记录
  - 处置结果反馈

### 数据保留策略
- **热数据**（近30天）：
  - 全量存储，支持秒级查询
  - 存储介质：SSD/内存数据库

- **温数据**（31-90天）：
  - 压缩存储，支持分钟级查询
  - 存储介质：HDD/对象存储

- **冷数据**（91天-3年）：
  - 归档存储，支持小时级查询
  - 存储介质：对象存储/磁带库

- **销毁策略**：
  - 超过保留期自动销毁
  - 销毁前审批流程
  - 销毁记录永久留存

### 脱敏策略
- **静态脱敏**：
  - 手机号：138****1234
  - 身份证：330102********1234
  - IP地址：192.168.*.* 或 192.168.1.*

- **动态脱敏**：
  - 根据用户角色动态脱敏
  - 高权限用户可查看明文
  - 低权限用户自动脱敏

- **查询脱敏**：
  - 支持模糊查询但返回脱敏结果
  - 精确查询需更高权限

## 8. 典型场景案例

### 场景一：勒索病毒防御与应急
**威胁**：勒索病毒通过钓鱼邮件入侵，加密文件勒索

**检测**：
- 邮件网关检测到钓鱼邮件（规则命中）
- 终端EDR检测到异常进程行为
- 文件系统监控发现大量文件被加密

**关联分析**：
- 时间窗内关联：邮件打开→进程启动→文件加密
- 受影响主机：3台服务器+15台终端
- 攻击路径：邮件→执行→横向移动→加密

**自动处置**：
1. 隔离受感染主机（断网）
2. 终止恶意进程
3. 禁用相关账号
4. 通知安全团队+创建P0工单
5. 启动备份恢复流程

**价值**：从发现到隔离<5分钟，避免进一步扩散

### 场景二：内部数据泄露调查
**威胁**：员工违规下载大量敏感数据并外传

**检测**：
- DLP检测到大量文件下载（统计异常）
- 网络流量分析发现异常外发
- 终端审计发现U盘拷贝行为

**关联分析**：
- 用户：张三（研发部门）
- 时间：下班后22:00-02:00
- 数据：客户数据库+源代码
- 动作：数据库导出→文件压缩→U盘拷贝→邮件外发

**处置**：
1. 立即禁用账号
2. 锁定U盘（远程删除数据）
3. 通知HR+法务
4. 保全证据（日志+录像+磁盘镜像）
5. 生成完整时间线报告

**价值**：完整证据链，支持后续法律诉讼

### 场景三：Web应用攻击防护
**威胁**：黑客通过SQL注入攻击获取数据库权限

**检测**：
- WAF检测到SQL注入特征
- 数据库审计发现异常查询
- 应用日志发现错误注入

**关联分析**：
- 攻击源：境外IP（威胁情报命中）
- 攻击手法：SQL注入+文件上传+Webshell
- 受影响：2个应用系统

**自动处置**：
1. WAF自动封禁攻击IP
2. 应用系统降级保护（临时关闭注入点）
3. 数据库账号临时禁用
4. 通知开发团队修复漏洞
5. 生成漏洞修复工单

**价值**：攻击自动阻断，业务影响<2分钟
